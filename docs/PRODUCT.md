# 基金收益预测助手 V2

> 全栈基金投资智能分析平台 | Flask + AI + 量化引擎

---

## 一、产品概述

**基金收益预测助手**是一款面向个人投资者的智能基金分析工具，集实时行情、量化信号、AI 对话、智能推荐于一体。系统通过多数据源融合、多因子量化模型和 AI 多模态能力，帮助用户做出更理性的投资决策。

**技术栈：** Python Flask | 原生 JavaScript | Chart.js | OpenAI-Compatible API | 多源金融数据 API

## 三、核心功能模块

### 模块 1：实时持仓管理

- 添加/删除基金持仓（输入基金代码自动补全名称）
- 批量导入：支持**文本粘贴**（自然语言/JSON）和**图片识别**（AI Vision）两种方式
- 资产总览面板：预估总资产、今日预估收益、累计收益、持仓数量、收益率
- 持仓数据本地持久化（localStorage）
- 一键导入任意格式的持仓数据（正则 + AI 双引擎解析）

### 模块 2：大盘行情监控

- 上证指数 / 深证成指 / 创业板指实时行情
- 成交量、成交额、振幅等详细数据
- 交易时间自动检测（开盘/休市状态指示灯）
- 热门板块横向滚动展示（涨跌幅 + 领涨股）

### 模块 3：基金深度分析

- 实时净值估值（30 秒级更新）
- 历史净值走势图（Chart.js，支持多时间窗口）
- 重仓股持仓明细（股票代码、名称、占比、实时涨跌）
- 多周期收益展示（近 1 月 / 3 月 / 6 月 / 1 年）
- 基金详情弹窗（净值曲线 + 重仓股 + 信号 + 收益）

### 模块 4：多因子买卖信号

- 5 维度因子综合评分，输出 0-100 分买/卖信号
- 信号等级：强烈建议买入 / 建议买入 / 观望 / 建议卖出 / 强烈建议卖出
- 每只基金可展开查看因子明细与评分依据

### 模块 5：智能基金推荐

- 多维基金池采集（股票型 / 混合型 / 全市场，6 个月 / 1 年收益排名）
- 两阶段评分：快速筛选 → 深度分析
- 推荐分级：强烈推荐 / 推荐买入 / 值得关注
- 每只推荐基金附带综合评分与多因子分析详情

### 模块 6：AI 智能助手

- 悬浮聊天面板，支持连续对话
- 流式输出（SSE 协议），打字机效果
- 图片上传识别（Vision API，可识别持仓截图）
- 专业基金投资顾问人设，中文回复

### 模块 7：附加功能

- **贵金属行情**：COMEX 黄金/白银实时价格，人民币/克自动换算，K 线走势图
- **价格提醒**：设置基金涨跌幅阈值，自动检测触发
- **市场指数弹窗**：点击查看详细行情（开盘/最高/最低/成交量/成交额）

## 四、技术亮点与创新

### 亮点 1：多因子量化信号引擎

**创新点：** 将专业量化投资方法论落地为可交互的买/卖信号系统。

系统内置 5 个独立因子，每个因子输出标准化评分（0-100），再按权重合成最终信号：

| 因子 | 计算方法 | 权重 | 意义 |
|------|----------|------|------|
| MA 均线位置 | 价格 vs MA20/60/120/250 偏离度 | 30% | 趋势方向 |
| RSI(14) | 14 日相对强弱指标 | 20% | 超买/超卖 |
| 近期动量 | 5/10/20 日收益率 | 15% | 短期动能 |
| 回撤幅度 | 当前净值 vs 历史高点回撤比 | 15% | 风险释放程度 |
| 历史分位 | 当前净值在全部历史数据中的百分位 | 20% | 估值水平 |

**核心代码路径：** `quant/signals.py` → `calculate_signal()` 函数（约 180 行）

**参考方法论：** Morningstar 评级、Lipper Leader 评分体系。

---

### 亮点 2：两阶段智能推荐引擎

**创新点：** 通过「快速评分 + 深度分析」两阶段设计，在精度与性能之间取得最优平衡。

**Phase 1 — 快速筛选（0 额外 API 调用）：**
- 从东方财富排行榜 API 采集 200+ 基金候选池（覆盖股票型/混合型/全市场 × 6 月/1 年排名）
- 仅用排名数据中的多周期收益做快速评分
- 评分因子：加权收益（60%）+ 正收益一致性（25%）+ 趋势加速度（15%）
- 排序后取 Top 45 进入下一阶段

**Phase 2 — 深度分析（并发执行）：**
- 使用 `ThreadPoolExecutor`（max_workers=10）并发获取每只基金的详细数据
- 综合评分 = 收益能力（30%）+ 风险控制（20%）+ 夏普比率（20%）+ 收益一致性（15%）+ 技术面（15%）
- 风险控制因子：年化波动率 + 最大回撤（基于日收益率标准差）
- 夏普比率：简化版（无风险利率假设为 0），年化处理
- 最终按百分位分布划分推荐等级（Top 18% 强烈推荐，18%-55% 推荐买入，55%-85% 值得关注）

**核心代码路径：** `services/recommend_service.py` → `fetch_fund_pool()` + `quick_score()` + `calculate_comprehensive_score()` + `get_recommendations()`

---

### 亮点 3：AI 多模态能力集成

**创新点：** 将大语言模型的视觉理解能力与传统正则解析结合，实现「截图即导入」的极致体验。

**能力矩阵：**

| 能力 | 实现方式 | 场景 |
|------|----------|------|
| 图片识别导入 | Vision API（OpenAI 兼容） | 上传 App 持仓截图，自动识别基金代码/名称/金额/收益 |
| 文本智能解析 | 正则 + AI 双引擎 | 从任意格式文本中提取基金信息（支持自然语言、JSON、混合格式） |
| AI 对话助手 | SSE 流式输出 | 专业基金投资顾问，支持连续对话，上下文记忆 |
| 图片对话 | Vision + Chat 融合 | 在 AI 聊天中发送图片，识别内容并回答问题 |

**文本解析策略（正则 + AI 双引擎）：**
1. 先尝试 JSON 解析（结构化数据）
2. 逐行正则提取基金代码（`\b\d{6}\b`）、中文名称、金额
3. 代码/名称可只填其一，系统自动补全（本地基金列表 + 在线搜索 API）
4. 正则失败时回退到 AI 语义理解

**流式输出实现：**
- 后端：SSE（Server-Sent Events）协议，逐 chunk 推送
- 前端：`EventSource` / `fetch` + `ReadableStream`，打字机渲染效果
- 支持 `X-Accel-Buffering: no` 以兼容 Nginx 反向代理

### 亮点 4：多源数据融合与智能容错

**创新点：** 三数据源互为备份，单一数据源故障时自动降级，保障服务高可用。

**数据源架构：**

```
东方财富（主力）          新浪财经（补充）          东方财富搜索 API（兜底）
├── 基金估值              ├── LOF/ETF 实时行情      ├── 基金名称模糊搜索
├── 历史净值              ├── 大盘指数              └── 基金代码/名称互查
├── 重仓股持仓            ├── 贵金属实时价格
├── 基金排行榜            └── USD/CNY 汇率
└── 板块行情
```

**容错策略：**
- 基金估值：东方财富失败 → 自动回退到新浪财经股票 API（支持 LOF/ETF）
- 市场前缀自动判断：`16xxxx` → 深圳 LOF，`5xxxxx` → 上海 ETF
- 基金搜索：本地列表匹配 → 东方财富在线搜索 API

**多层缓存策略：**

| 数据类型 | 缓存 TTL | 原因 |
|----------|----------|------|
| 基金估值 | 30 秒 | 交易时段频繁变动 |
| 市场指数 | 30 秒 | 实时性要求高 |
| 贵金属价格 | 60 秒 | 国际市场波动 |
| 热门板块 | 120 秒 | 中等更新频率 |
| 历史净值 | 300 秒 | 日级数据，低频变动 |
| 重仓股持仓 | 300 秒 | 季报级数据 |
| 基金推荐 | 600 秒 | 计算成本高，适度缓存 |

---

### 亮点 5：工程化设计

**创新点：** 轻量级技术栈下的工程化最佳实践。

**令牌桶限流器（`ratelimit.py`）：**
- 多 key 独立配置（eastmoney: 5/s, sina: 3/s）
- 线程安全（`threading.Lock`）
- 阻塞等待 + 超时机制
- 支持装饰器和手动 `acquire()` 两种调用方式

**并发推荐引擎：**
- `ThreadPoolExecutor`（可配置 max_workers，默认 10）
- `as_completed` 超时控制（默认 45 秒）
- 单基金评分失败不影响整体结果

**前端架构：**
- 零框架依赖（原生 JS，IIFE 封装，总代码 ~71KB）
- Chart.js 可视化（净值走势图、贵金属 K 线）
- 响应式布局（移动端适配，`viewport` meta + CSS 媒体查询）
- 涨跌色彩体系（红涨绿跌，CSS 类名语义化：`up` / `down` / `flat`）
- 本地存储持久化（持仓数据 `localStorage`）

**CLI + Web 双入口：**
- Web：`python app.py` → 浏览器访问 `localhost:5000`
- CLI：`python cli.py list` / `add` / `signal` / `estimate`
- 一键启动：`start.bat` 自动开浏览器 + 启动服务
